# D5 Light Visual Direction — 浅岩台 / Mineral Shelf

## 命题

`浅岩台`把 Tetris 放回一个明亮、安静、可立即操作的游戏台面：画面的重心始终是居中的 10 × 20 轻矿物棋盘，而不是标题、侧栏或装饰性信息。识别手势是棋盘、Next 和按键轨共用的**双层矿脊脚**：底缘一条薄赭石、右缘一条蓝灰，像托起轻矿物台面的两片错位薄板。

这是一种可实现的 CSS/DOM 视觉合约，不是位图资产。D5 不使用 imagegen、外部纹理、图片文字或生产 App/CSS。

## 研究原则的转译（非复制）

只读参考的个人网站提供了三个方法：一个可辨识手势、受控的信息密度、以及让空白服务于主内容的排版。D5 的转译是：

- 一个手势：双层矿脊脚只服务棋盘和输入系统；不使用原站的点阵、青绿色、符号、网格、header/侧栏或字体组合。
- 受控密度：ready 只显示三个完整模式名和当前被选模式的一组目标/结束规则；局内只保留当前模式、唯一 Next、模式所属统计和输入帮助。
- 有方向的留白：桌面把 612px（68vh）的棋盘置于视轴；portrait 把信息排成一列，避免把文字塞进棋盘侧缝。

## Token

| 类别 | Token | 值 / 规则 |
| --- | --- | --- |
| 纸面 | `--paper` | `#F7F4EC`，纯暖纸；页面无重复线、点或坐标纹理 |
| 棋盘 | `--mineral` / `--mineral-hi` / `--mineral-deep` | `#E1E5DC` / `#FDFCF8` / `#C7CFC3`；轻矿物表面与浅层托影 |
| 墨色 | `--ink` / `--muted` | `#31413E` / `#6D7A74`；文字、边界、低优先级说明 |
| 矿脊 | `--signal` / `--reply` | `#C56E4F` 薄赭石底脚、`#6D879B` 蓝灰右脚；只作回应与结构脚缘 |
| 方块 | `--piece-a…d` | 有限印刷油墨：赭石、蓝灰、橄榄、灰紫；非彩虹、非霓虹 |
| 字体 | `--display` | `Iowan Old Style`, `Source Han Serif SC`, `Noto Serif SC`, serif；只用于小型 `Tetris` 与状态标题 |
| 字体 | `--body` / `--mono` | 系统中文无衬线；`Cascadia Mono` 仅用于数值 |
| 间距 | 4 / 6 / 8 / 10 / 12 / 16 / 24px | 模块至少 8px；portrait 游戏簇两侧至少 16px |
| 边框 | 1px | 深墨或低对比矿物线；零圆角 |
| 阴影 | `7px 7px 0 mineral-deep` + `12px 12px 0 mineral-hi` | 只用于棋盘与双层矿脊的薄层错位，不做漂浮卡片 |
| 动效 | 140–180ms `ease-out` | 仅按钮 focus/active、模式 marker 与暂停纸带；`prefers-reduced-motion` 时关闭位移和过渡 |

## 布局与信息语法

- 小型完整 `Tetris` 是唯一 header 内容；当前模式永远由 board 上方的 context 行或 mode rail 承担。
- ready 使用平铺的三等分文字 rail（马拉松模式 / 竞速模式 / 解谜模式），仅 selected marker 为一段矿物+赭石下划线；没有分隔单元、胶囊或卡片。
- 运行中只有一个 `下一个方块`；不出现 Hold / 暂存。
- 输入是一个五区连续印刷轨：`← 左移`、`→ 右移`、`↑ 旋转`、`↓ 快速下落`、`⇣ 直接落底`；每区至少 44px。
- 暂停是棋盘内部的窄纸带；页面其余区域不覆暗。模式切换和解谜关卡选择在棋盘外的扁平行中完成。

## 禁止项

- 深色棋盘井、巨大标题、斜向色带、松散左 rail、空白文学文案、假 telemetry。
- 玻璃、dashboard、卡片阵列、圆角胶囊、右下浮层、赛博霓虹、页面网格壁纸。
- `T.`、模式缩写、`Hold`、`暂存`、无来源分数/速度词、设计概念标签。
- 用生成图、截图文字或外部生产资产掩盖 HTML/CSS 布局问题。
